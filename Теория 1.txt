ООП - это парадигма программирования, в которой программы структурируются как коллекции объектов, каждый из которых представляет собой комбинацию атрибутов и методов и строится на четырёх ключевых принципах: инкапсуляция, полиморфизм, наследование и абстракция.
ООД - разделении задачи на более малые. Делает код более читаемым, масштабируемым, помогает делать лучшее покрытие тестами.
Дизайн ПО - процесс создания проекта ПО рассматривая структуру модулей, организацию внутри модулей и взаимодействие между объектами. Пример: Объекты - персонажи игры; Модули -  игровой движок, графический движок, Структуру модулей - Графический движок зависит от Игрового; Организация внутри модуля - структура кода, такая как класса методы функции и т. д. Взаимодействие - как персонажи между собой взаимодействуют.
Число Миллера 7 +-2, а ещё лучше 5 +- 2. Сколько человек может держать элементов в кратковременной памяти. Примеры: количество методов public, количество аргументов функции и методов, размер enum, количество логических блоков в одной функции, длинна цепочки вызовов методов, глубина (if/while/for/ switch-case).
5 признаков сложных систем по Гради Бучу: Иерархическая структура, Относительность выбора элементарных компонентов, Разделение обязанностей, Общие шабоны, Стабильные промежуточные формы. Пример: 
Среда разработки игр
Иерархическая структура: движок - сцены - объекты - скрипты
Относительность выбора элементарных компонентов: для движок эл. компон. - сцены для сцены эл. компон. - объекты для объекты эл. компон. - скрипты;
Разделение обязанностей: есть отдельные компоненты для физики, логики, отрисовки, звуков.
Общие шабоны: например Singleton
Стабильные промежуточные формы: компилируемые и тестируемые версии движка с новыми фичами
